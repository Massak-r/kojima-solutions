# Kojima Solutions — Full App Audit & Improvement Roadmap

> Generated 2026-04-24. Scope: UX, UI, features, architecture, data layer.
> Companion doc: `client-portal-ux-insights.md` (client-facing UX principles).

---

## 0. Executive summary

Kojima Solutions is a mature solo-consultant operating system: intake → brief → cadrage → modules → steps → quote → invoice, plus sprint/objective workspace, personal treasury, admin documents, client portal. The visual language (zen/washi palette, glass-card, Sora/Inter) is strong and consistent. Landing-page and stakeholder-facing pages feel premium.

Four structural problems dominate, in decreasing order of blast radius:

1. **Data sync debt between workflow surfaces** — intake, brief, cadrage, modules, quote, invoice each own their own copy of overlapping data. Edits don't propagate; regeneration is a 4-place ritual. This is the biggest source of daily friction for a solo operator.
2. **State-management fragmentation** — `@tanstack/react-query` is installed and wrapped around the app but **never used** (0 `useQuery`/`useMutation` call sites). Every context reinvents: localStorage-first + silent `.catch(() => {})` API sync, no rollback, no error surface to the UI. Optimistic updates can desync forever.
3. **Monolithic pages** — `ClientDashboard.tsx` (1866 lines), `Tresorerie.tsx` (1161), `IntakeForm.tsx` (1148), `KojimaSpace.tsx` (~1100), `Accounting.tsx` (977), `SprintPage.tsx` (824), `AdminSpace.tsx` (892). Each mixes state, fetch, UI, and calculation. They block safe iteration.
4. **Navigation/affordance fog** — five competing entry points (Header, BottomNav, CommandPalette, QuickActionFAB, FloatingCTA) with unclear hierarchy; keyboard shortcuts undiscoverable; legacy route redirects reveal unresolved IA decisions.

Everything below is ordered so the early sections compound: fix #1 and #2 once, and the pages in #3 shrink naturally.

## 1. Workflow data flow — the core friction

### 1.1 Intake → Brief → Cadrage → Modules → Quote doesn't propagate

| Where it's entered | Where it's re-entered | Today's behaviour |
|---|---|---|
| Intake budget / tier | `ProjectCadrage.budgetValidated`, `ProjectModules` complexity seed | Frozen at conversion; no sync back |
| Intake features / goals | Brief display, Cadrage deliverables | Read-only in Brief; not seeded into Cadrage |
| Modules + complexity | Cadrage deliverables, Funnel phases, Steps tasks, Documents quote lines | Four independent generator functions (`generateDeliverablesFromModules`, `generateQuoteLinesFromModules`, `generatePhasesFromModules`, `generateTasksFromModules`). Each re-import is manual. |
| Quote line | Invoice line | Manual conversion; no `sourceModuleId` / `sourceComplexity` back-reference → disputes hard to defend |

**Fix pattern.** Introduce a `ModuleResolver` (one class, one source of truth) that exposes `.toDeliverables()`, `.toPhases()`, `.toTasks()`, `.toQuoteLines()`. Each consumer reads the resolver instead of re-running a local generator. Downstream objects keep a `sourceModuleId` + `sourceComplexity` so the chain is traceable both ways.

**Immediate win**: seed `ProjectCadrage.budgetValidated` from `intake.budget` on first load (editable), and pre-fill quote `clientId` / `currency` from the brief.

### 1.2 Project sync state is invisible

There is no single indicator of "this project is synced across brief/cadrage/modules/quote." A `ProjectStepNav` completion indicator (see §2.4) solves the visual half; the resolver above solves the data half.

### 1.3 Quote → invoice handoff is hidden

`ProjectDocuments.tsx:293-300` renders the "convert to invoice" button **only** when the quote is validated and only on hover on desktop. Mobile users have no path. There's also no "invoicing due" dashboard card. For a solo consultant this is missed-revenue risk.

**Fix**: render a permanent action on the quote row (icon + label), add a "À facturer" card to `Dashboard` / `KojimaSpace` showing validated quotes older than N days.

### 1.4 Three share-link types, no clarity

Portal slug vs. stakeholder token vs. funnel token — three overlapping mechanisms documented only as a footnote in `ProjectFunnel.tsx:697-732`. Ship a single "Partage" modal per project that shows all active links, a one-line "Who sees what" per link, and a "copy all" action.

### 1.5 Objectives ↔ projects ↔ clients is declared but not enforced

`objective.linkedProjectId` / `linkedClientId` exist in `src/api/objectiveSource.ts:43-56` but are cast via `(o as any)`. No DB foreign key; no cascade on delete; no validation that the link target belongs to the current user; no orphan cleanup. From the workspace UI, the link is read-only — no unlink/relink.

**Fix**: add backend FK + validation, typed mapper (remove `as any`), inline unlink/relink on the objective header.

## 2. Core surfaces — specific findings

### 2.1 Sprint page (`src/pages/SprintPage.tsx`, 824 lines)

- **Radical focus mode has no exit.** The trigger is a tiny `text-muted-foreground underline` link (line ~352). Once engaged there's no persistent chrome and no `Esc` handler. Add a sticky "← Retour" button and bind `Esc`.
- **Weekly-review dialog loops.** Fires on Friday ≥14:00, dismissed via localStorage. Closing at 13:59 and reopening at 14:05 re-shows it. Use `ISO-WEEK-ID:seen` as the key.
- **LocalStorage polling every 4 s** (lines ~205-215) to detect a focus session from another tab. Replace with the `storage` event or `BroadcastChannel`.
- **Daily-commit gate is one-way.** Once skipped, re-flagging items later in the day does not re-open the commit dialog. Add a manual "Commit du jour" entry point in the header that's always available.
- **Map/sort rebuilt every render.** `sorted` (lines ~250-264), `objectiveTextById`, `subtaskById` (lines ~221-230, 266-270). Wrap in `useMemo`.
- **Stale-item threshold** (7 days) is hardcoded with no tooltip explaining the badge. Either make it configurable in Settings or add a hover explanation.
- **Recurring items have no visual distinction** in the week view — they use the same chip as scheduled items.

### 2.2 Objective workspace (`src/pages/ObjectiveWorkspace.tsx`)

- **Shortcuts `F / N / ? / Esc` are undiscoverable.** Add a persistent "?" pill in the header.
- **Inline title edit saves on blur** with no confirmation — add a subtle checkmark or toast.
- **Optimistic subtask create** (lines 163-172) has no rollback on failure — the temp item stays. Either rollback + toast, or retry with exponential backoff.
- **Subtasks are rendered from two sources** (local optimistic + reload). Unify via react-query (see §3).
- **Sub-subtasks don't look nested** — no indentation, same chrome as top-level.
- **`listSessions()` refires on every window focus** — debounce or skip while a session is running.
- **Tabs lose scroll position** on switch (Notes ↔ Files). Store scroll offset in a keyed ref.

### 2.3 Projects Dashboard (`src/pages/Dashboard.tsx`)

- **Kanban is mouse-only.** No keyboard column-move, no right-click menu, no bulk selection. The drag handle appears only on hover (`md:opacity-0 md:group-hover:opacity-100`) so mobile users don't discover it at all.
- **Silent drag failures** — a failed status update (line ~101) snaps back without a toast.
- **Search doesn't highlight** matches in results; for >20 projects this is navigation tax.
- **`rotate-2` on drag** feels playful vs. the rest of the palette — drop to `rotate-1`.
- **DragOverlay re-renders full ProjectCard every frame** — wrap in `React.memo`.

### 2.4 Project-step pages (Brief / Cadrage / Modules / Steps / Documents)

- **`ProjectDetailsPanel` is buried in a collapsible section** below the fold (`ProjectBrief.tsx:113-115`). Client/status/dates should be visible on every step page. Promote to a sticky sidebar (matching `ProjectModules`'s left panel) or a horizontal summary strip under the title.
- **`ProjectStepNav` has no completion indicator.** Add `✓` or `3/7` per tab so "Where am I in this project?" is answerable at a glance. Complements §1.2.
- **`ProjectSteps` sidebar collapses on mobile** (line 338-347) stacking TaskFormPanel above the timeline. On a 375 px phone you can't see both. Convert to a `Sheet` drawer on mobile, keep split on ≥md.
- **Module complexity UX is vague.** Simple/Advanced/Custom tooltips say "configuration standard" vs "personnalisation poussée" without showing the price delta. Show per-module price per complexity inline (e.g. "Simple 8 000 · Advanced 15 400 CHF").
- **Cadrage has no empty-state copy** — loading spinner then blank. Add contextual "Start in Modules →" hint.
- **Unused imports in `App.tsx`** (`ProjectTasks`, `ProjectFunnel`, `ProjectDetails`). Ship cleanup.
- **Nine legacy route redirects** (`App.tsx:125-134`) point to 2-3 pages. Document and remove once any bookmarked URL is 301-handled server-side.

### 2.5 KojimaSpace hub

- **Single-page overload.** Objectives + projects + invoices + personal costs + calendar + analytics + email queue + intake on one scroll. For a hub, tabs or a 2-column stream layout beats a single stream.
- **`hiddenCats` toggle is localStorage-only with no UI.** If Famille gets hidden by accident there's no recovery without DevTools. Expose a "Afficher toutes les catégories" toggle in the header.
- **Project status vs. payment status colours both use amber** for semantically different states. Split the palette.
- **No "flag top 3 for today"** from the hub — the core value prop of a daily dashboard is missing.

### 2.6 Portfolio

- **Placeholder letter is low-contrast** (`text-primary/20`) on light backgrounds.
- **No filter by client/type/date**, no case-study drill-down, no image `onError` fallback.
- **Date format breaks on mobile** (wraps across two lines). Switch to `MM/YYYY – MM/YYYY` or a duration.

### 2.7 Accounting / Trésorerie

- **Dual state management for expenses.** `loadExpenses()` from localStorage + `listExpenses()` from API (`Accounting.tsx:99-100`; `Tresorerie.tsx:26-41`). Race condition; silent API failures; no "last synced" indicator. Pick one source, route through react-query.
- Both files are in the 900–1200 line range; extract `BudgetTab`, `ConsumableManager`, `ExpenseTable`, `StatCard` containers.

### 2.8 Admin space & settings

- **No breadcrumb** in the folder manager (`AdminSpace.tsx`). `getFolderPath()` exists — use it in a breadcrumb header.
- **Drag-drop in dnd-kit has no keyboard path** for folder reordering.
- Settings is reachable only via the header overflow menu on mobile — promote to BottomNav or add a visible desktop entry.

### 2.9 Client-facing pages

- **ClientDashboard has no card → project drill-down**; clicking a project card does nothing. Add navigation.
- **QuotesList has no client filter** — context-switching tax when you need "all quotes for Acme."
- **Client email gate re-asks every visit.** No opaque session token. This is both a security fix (see §4) and a UX win.

## 3. State / data layer — the quiet crisis

### 3.1 Adopt react-query (it's already installed, just unused)

0 call sites of `useQuery`/`useMutation` in `src/`. `QueryClientProvider` is wrapped around the app for nothing. Every context re-implements caching with localStorage + silent-catch. This is the single highest-leverage refactor in the app.

**Migration order (low → high risk):**
1. `useObjectives` + `useSubtasks` (already relatively isolated)
2. `ClientsContext`
3. `ProjectsContext`
4. `useQuotes`
5. Keep `LanguageProvider`, `CompanySettingsProvider`, `AuthProvider` as contexts — they are not API-backed.

### 3.2 Stop swallowing errors

8+ instances of `.catch(() => {})` across `ProjectsContext`, `ClientsContext`, `applyAndSync`, `pushNotifications`. Every mutation that "succeeds" on the UI may have failed on the server with no surface. Once on react-query, `onError` becomes a natural home for toasts + rollback.

### 3.3 Optimistic updates need rollback

`applyAndSync` (`ProjectsContext.tsx:80-92`) mutates local state + persists before the API resolves, and never undoes on failure. Pattern: snapshot previous state, run mutation, rollback if `onError`, invalidate on `onSettled`.

### 3.4 Remove `(o as any)` casts

`src/api/objectiveSource.ts:43-56` casts to `any` to read fields that the generated types don't know about (`definitionOfDone`, `linkedProjectId`, `linkedClientId`). Fix: extend the generated type, or define a Zod schema at the API boundary and let TS infer from it.

### 3.5 Offline queue needs a retry cap + ordering

`src/lib/offlineQueue.ts:51-81` retries failed requests forever (any 4xx will loop until manually cleared), preserves no ordering between DELETE/POST, and has no conflict detection. Add: `retryCount`, `maxRetries`, 4xx → dead-letter, FIFO by enqueue time, per-resource ETag support.

### 3.6 Context provider stack

`App.tsx:72-82` nests 6 providers. After the react-query migration, `QuotesProvider`, `ClientsProvider`, `ProjectsProvider` collapse into query hooks and the tree shrinks to `Auth + Language + CompanySettings`.

## 4. Security

1. **Admin password hardcoded** as `"kojima2025"` fallback (`src/lib/auth.ts:2`). Ship an env-only flow; fail closed if `VITE_ADMIN_PASSWORD` is missing.
2. **Client-side auth by string compare** — `localStorage.getItem(ADMIN_KEY) === "authenticated"`. Trivial DevTools bypass. Replace with a server-issued session cookie (HttpOnly, SameSite=Lax) + server-side validation on every admin PHP endpoint.
3. **Client portal identity = email in localStorage.** PII, not a secret, and no rate limit on `client_login.php`. Issue an opaque token server-side; rate-limit by IP.
4. **No CSRF protection on admin endpoints.**
5. **Auto-migration anti-pattern** — many PHP endpoints open with `try { $pdo->exec("ALTER TABLE …"); } catch (Throwable) {}`. Hides failures and makes schema state opaque. Move to a real migration tool (Phinx) or at minimum log the catch and version-gate migrations.
6. **File-upload `validateMagicBytes`** exists in `_bootstrap.php:107-130` but I don't see it invoked from upload endpoints. Verify `admin_files.php`, `personal_files.php`, `objective_files.php`, `upload.php` all route through it. Add explicit size limits.
7. **`(o as any)` + no Zod** at the API boundary means a bad server response can write arbitrary shapes into state.

## 5. Accessibility & mobile

- **~75 ARIA attributes total** across ~163 components. Priority: `aria-current="page"` in `BottomNav`, `aria-label` on `NotificationBell` / `QuickActionFAB` / `CommandPalette` dialog, `aria-live` on toast/banners.
- **No focus restoration on route change** — `PageTransition.tsx` doesn't move focus. Add `main.focus()` after animation completes.
- **`pb-safe-bottom` audit** — only `KojimaSpace` applies it. AdminSpace, Settings, Tresorerie, Quotes all risk hiding their last row behind the bottom nav on iOS/Android. Move the padding into `AdminContentWrapper` in `App.tsx`.
- **UpdateBanner requires `isAdmin && isPWA`** — loosens not needed, but the `isPWA` gate means a user who uses the web version won't know to refresh. Drop the `isPWA` gate.
- **FAB and UpdateBanner both live at `bottom-24` on mobile** — stack collision. Coordinate z-index and spacing.

## 6. Design system

Strong foundations — glass-card, zen palette, Sora/Inter — but three gaps:

1. **No documented type scale.** Pages use ad-hoc `text-3xl md:text-4xl`, `text-[10px]`, etc. Define `h1…h6 + body + caption` as Tailwind tokens and refactor inward.
2. **Card variants drift** — `glass-card`, `bg-card border-border rounded-xl`, `bg-secondary`, even `bg-white` in places. Pick one primary card and one glass variant; delete the rest.
3. **Empty state component exists but is barely used.** Apply `<EmptyState>` consistently on Dashboard columns, Sprint backlog, AdminSpace, Portfolio, etc.
4. **Status-colour inconsistency across pages.** Completed = sometimes emerald, sometimes palette-sage. Define `STATUS_COLORS` once and import everywhere.

## 8. Performance

- **Virtualize the Kanban** (Dashboard) once projects > 30 per column — `react-window` is already viable with the existing card shape.
- **Memoize `ProjectCard`** for DragOverlay.
- **Lazy-route the heavy pages** — `ClientDashboard`, `Accounting`, `Tresorerie`, `IntakeForm` are 40–90 kB each and eagerly imported in `App.tsx`. `React.lazy` + `Suspense` + route-level code-split. Biggest bundle-size win in the app.
- **Debounce viewport resize** in SprintPage's dynamic grid compute.
- **Shared fetch** — 6 contexts each fire on mount. After react-query migration this collapses to deduped queries.
- **Sharp + html2canvas + jspdf + mp4-muxer** are client-bundled; confirm tree-shaking or move PDF/video generation server-side.

---

## 9. MCP server

- **No caching** — every Claude tool call is a fresh HTTP hop to the public API (`tools/mcp-server/src/api.ts:39-55`). Add a short-TTL in-memory cache keyed by path+query.
- **No auth beyond `X-API-Key`** — if/when the MCP server runs in a shared environment, add request signing.
- If the MCP server co-locates with the PHP host, consider direct DB access for read-only tools — cuts latency and avoids HTTP overhead.
